```markdown
# 组件UI优化规范 (Component UI Optimization Protocol)

## 一、设计基准参数 [可根据项目调整]

### 字体系统
| 层级 | 尺寸 | 权重 | 用途 |
|-----|------|------|------|
| H1 | text-lg (18px) | semibold | 页面主标题 |
| H2 | text-base (16px) | medium | 卡片/区块标题 |
| Body | text-sm (14px) | normal | 正文内容 |
| Caption | text-xs (12px) | normal | 辅助说明、标签 |

### 间距系统
| 类型 | 数值 | 应用场景 |
|-----|------|---------|
| 紧凑 | gap-1, p-2 | 内联元素、标签组 |
| 标准 | gap-2, p-3 | 表单项、列表项 |
| 宽松 | gap-3, p-4 | 卡片、区块容器 |
| 分隔 | gap-4, p-6 | 页面级区块间 |

### 布局密度
- 应用类型：工具型应用 → 信息密度优先
- 单屏利用率：>70%
- 最大空白区域：不超过 48px

## 二、扫描检查清单

执行组件优化时，依次检查：

### 2.1 字体问题
- [ ] 是否存在 text-xl/text-2xl 以上的过大字体
- [ ] 同一层级文字字号是否统一
- [ ] 是否有未使用设计基准的自定义字号

### 2.2 间距问题  
- [ ] 是否存在 p-6/p-8 以上的过大内边距
- [ ] 是否存在 gap-6/gap-8 以上的过大间隙
- [ ] 是否有不必要的 margin 造成空白堆积
- [ ] 嵌套容器是否有重复 padding

### 2.3 布局问题
- [ ] 是否有可合并到一行的元素分行显示
- [ ] flex 容器是否正确使用 gap 替代 margin
- [ ] 是否存在固定高度造成的空间浪费
- [ ] 响应式断点是否合理

### 2.4 一致性问题
- [ ] 与相邻组件的视觉风格是否协调
- [ ] 圆角、阴影是否符合项目统一标准

## 三、修复执行标准

### 字体修复映射
```
text-3xl → text-lg
text-2xl → text-base  
text-xl  → text-base
text-lg  → text-sm (如果是正文)
```

### 间距修复映射
```
p-8/p-10  → p-4
p-6       → p-3 或 p-4
gap-8     → gap-4
gap-6     → gap-3
m-* 分散  → 改用父容器 gap
```

### 布局修复原则
```
垂直排列可水平 → 改为 flex-row + gap-2
多层嵌套 padding → 合并到最外层
固定大高度 → 改为 min-h 或自适应
